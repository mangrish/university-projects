433-371 Interactive Systems Design
Project Specification

The object of this project is for you to gain some experience in designing and implementing a graphical user interface (GUI). The project is due at 12 noon on Wednesday 18 October and is worth 30% of your mark for 371.

Background

You're all familiar with tutenrol, the program you've used numerous times to enrol in tutes. What we want you to do is design a nicer GUI that provides essentially the same functionality and implement it as an application in JavaSwing. We also want you to justify your design.

There's a skeleton implementation (done by Tim Dwyer--thanks, Tim) at:

http://www.cs.mu.oz.au/371/proj/371proj.jar
also available via
/home/stude/data/371/proj/371proj.jar
There's also a README file in the same directory, which you should look at. You're to use this skeleton implementation as a starting point for your own. (A real enrol program would have to contact some shared database. The skeleton implementation--and yours--accesses just a local XML file. This is done for simplicity, and doesn't really affect the user interface.)

Submission

You should submit using
submit 371 proj
a jar file called enrol.jar, including a Makefile with default target to make your application and a target run to run it. Your submission should also include a justification of your design, explaining your choice of layout and GUI components. This justification should be formatted in appropriate HTML and be equivalent in length about 2 to 4 printed A4 pages. You should create the HTML markup directly with a text editor, not using a WYSIWYG HTML editor (we want you to learn the HTML markup). The justification should be accessible via the help menu of your application.

Note

  • This is an individual project. Normal Departmental policies about cheating apply.

  • Late submissions will be accepted until 25 October, but will incur a penalty of 2 marks per day late. Any other extensions require explicit approval by the lecturer.

  • The project specification may require changes and clarification between now and the due date. Announcements will be made in lectures, via the subject webpage or via cs.371, as appropriate. It is your responsibility to be aware of any such announcements.
